HTML

  • Enter your HTML notes here:
  • The head element contains information abobut the webpage.
  • The body element represents the visible content shown to the user.
  • Meta-Element: helps the browser to render the page correctly.
    It is found in the head tags and is not visible when the user looks at the webpage. It also defines the metadata as the character set, description, keywords, author, and viewport.
  • Element: a type of content on a webpage.
  • Tags: Elements are created using tags.
  • Body: Contains the content that is shown to the user.
  • Title: This tag is defines the title for a webpage. It is required and is extremely important for Search Engine Optimization.
  • Base Element: Defines the base URL for a webpage.
  • Viewport Element: sets the default view for our webpage.
  • SRC Attribute: Defines the location, or source, of the image file.
  • ALT Attribute: Contains a text string that describes the appearance and functionality of the image. It is also important for screen readers and other assitive technology.
  • Footer Element: appears at the bottom of the page. Usually contains the author, copyright, contact, sitemap, and navigation.

GIT

  • Enter your GIT notes here:
  • Below are the steps for my Git Flow.
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: Adds the changed files in our working directory to the staging area of Git.
  • git commit -m: Commits the changes to our repository with a message. The message should be short and descriptive.
  • gi pull origin main: This pulls the base (main or master) branch into the feature branch.
    You should always do this before pushing up any code.
  • git push origin (name of the branch): pushes up all of the changes.
  • Merge the feature branch into the base branch in Github by doing the following:
    • Click the "Pull Requests" tab at the top.
    • Select the "New pull request" button.
    • Compare the two branches.
    • Once on the form, give the Pull Request a title and a short description of the changes.
    • Click the "Create pull request" button.
    • Click the "Merge pull request" button.
